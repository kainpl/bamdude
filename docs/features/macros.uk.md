# Макроси

Система макросів BamDude дозволяє вставляти власний G-code у ключових точках виконання друку, забезпечуючи автоматизацію для механізмів заміни пластин, управління корпусом та індивідуальних робочих процесів.

---

## :material-code-braces: Що таке макроси?

Макроси -- це фрагменти G-code, які автоматично вставляються у файли друку при відправці з черги. Вони виконуються при певних подіях протягом життєвого циклу друку.

---

## :material-lightning-bolt: Події

Макроси можуть спрацьовувати за такими подіями:

| Подія | Коли спрацьовує | Приклад використання |
|-------|----------------|----------------------|
| **Print Start** | Перед першим рядком G-code | Повернення осей, перевірка пластини, запуск вентилятора корпусу |
| **Print End** | Після останнього рядка G-code | Виштовхування пластини, вимкнення нагрівачів, запуск механізму заміни |

---

## :material-pencil: Редактор макросів

### Створення макросу

1. Перейдіть до **Settings > Macros** (або **Settings > Workflow > G-code Injection**)
2. Виберіть цільову модель принтера
3. Введіть G-code для подій start та/або end
4. Зміни зберігаються автоматично

### Редактор G-code

Вбудований редактор надає:

- **Підсвічування синтаксису** для G-code
- **Нумерацію рядків** для зручності
- **Налаштування за моделлю принтера** -- різні макроси для X1C та A1 Mini
- **Попередній перегляд** точок вставки в реальному часі

---

## :material-printer: Налаштування за моделлю принтера

Макроси налаштовуються для кожної моделі принтера. У налаштуваннях відображаються лише ті моделі, які ви підключили.

| Модель принтера | Start G-code | End G-code |
|-----------------|:------------:|:----------:|
| A1 Mini | Власний | Власний |
| X1 Carbon | Власний | Власний |
| P1S | Власний | Власний |

Це дозволяє застосовувати різну автоматизацію для різних типів принтерів.

---

## :material-playlist-play: Як працює вставка

Коли планувальник відправляє друк з увімкненими макросами:

1. Знаходить фрагменти G-code для моделі цільового принтера
2. Створює **тимчасову копію** 3MF із вставленими фрагментами
3. Завантажує модифіковану копію через FTP
4. Видаляє тимчасовий файл після завантаження

!!! info "Оригінальні файли не змінюються"
    Вставка ніколи не змінює файли архіву чи бібліотеки. Для завантаження створюється лише тимчасова копія.

---

## :material-toggle-switch: Увімкнення для окремого елемента черги

1. Відкрийте діалог **Add to Queue** або **Edit Queue Item**
2. Позначте **Inject auto-print G-code**
3. Збережіть -- елемент черги відображатиме зелену позначку **G-code**

---

## :material-swap-horizontal: Інтеграція з режимом заміни пластин

Макроси є ключовим компонентом [режиму заміни пластин](swap-mode.md). Макрос end G-code відповідає за виштовхування пластини, а start G-code забезпечує правильне позиціонування нової пластини.

---

## :material-lightbulb: Поради

!!! tip "Спочатку протестуйте"
    Завжди тестуйте G-code вручну на принтері перед використанням у макросах. Некоректний G-code може пошкодити принтер.

!!! tip "Тримайте просто"
    Макроси виконуються синхронно. Тривалий G-code (наприклад, інтенсивне нагрівання) затримає наступний друк.

!!! tip "Поєднуйте з розумними розетками"
    Використовуйте end-макроси для виштовхування пластини, а потім автовимкнення розумної розетки після охолодження для повністю автоматизованого робочого процесу.
